Postman으로 테스트하려면 서버를 먼저 띄운 다음, Postman에서 HTTP 요청을 보내면 됩니다.

터미널에서 프로젝트 루트에서 실행하세요:

```bash
make
./mini_db_server -d db -p 8080 -t 4 -q 64
```

서버가 켜진 터미널은 그대로 두고, Postman에서 요청을 만드세요.

**1. Health Check**

- Method: `GET`
- URL: `http://127.0.0.1:8080/health`
- Body: 없음

예상 응답:

```json
{
  "success": true,
  "service": "mini_db_server"
}
```

**2. INSERT 요청**

- Method: `POST`
- URL: `http://127.0.0.1:8080/query`
- Headers:
  - `Content-Type: application/json`
- Body 탭:
  - `raw`
  - `JSON`

Body:

```json
{
  "sql": "INSERT INTO users VALUES ('Alice', 20);"
}
```

예상 응답:

```json
{
  "success": true,
  "type": "insert",
  "affected_rows": 1,
  "generated_id": 1
}
```

`generated_id`는 이미 데이터가 있으면 `1`이 아닐 수도 있어요.

**3. SELECT 요청**

- Method: `POST`
- URL: `http://127.0.0.1:8080/query`
- Headers:
  - `Content-Type: application/json`
- Body:

```json
{
  "sql": "SELECT * FROM users;"
}
```

또는 방금 insert한 id로:

```json
{
  "sql": "SELECT * FROM users WHERE id = 1;"
}
```

주의할 점은 이 서버가 지금은 `SELECT`, `INSERT`만 지원하고, 요청 하나에 SQL 한 문장만 허용한다는 겁니다. 그래서 이런 건 에러 납니다:

```json
{
  "sql": "SELECT * FROM users; SELECT * FROM users;"
}
```

서버 끌 때는 서버 실행 중인 터미널에서 `Ctrl + C` 누르면 됩니다.
