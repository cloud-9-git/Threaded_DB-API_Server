# mini_db_server async 고동시성 스트레스 리포트

## 처음 보는 분을 위한 안내

- 이 문서는 수천 개 이상의 요청을 거의 동시에 보내는 극단 상황을 보는 보고서입니다.
- 여기서는 `성공`이 아주 높지 않아도 서버가 잘못 구현됐다고 단정하면 안 됩니다. 서버가 감당 가능한 만큼만 받고 나머지를 `503`으로 돌려주는 것도 정상 동작의 일부이기 때문입니다.
- `워커 수`는 동시에 실제 일을 처리하는 인원 수, `큐 크기`는 잠시 대기시켜 둘 수 있는 줄 길이라고 생각하면 됩니다.
- `오류`는 서버가 아니라 테스트를 돌린 컴퓨터나 네트워크 한계까지 포함할 수 있습니다. 특히 `20k` 같은 큰 동시 접속에서는 이 점을 따로 봐야 합니다.
- 가장 먼저 볼 곳은 `버스트 수준별 최적 조합` 표와 `오류 유형 요약` 표입니다.
