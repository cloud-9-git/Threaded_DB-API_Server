# mini_db_server 고동시성 스트레스 리포트

## 처음 보는 분을 위한 안내

- 이 문서는 짧은 순간에 요청이 한꺼번에 몰릴 때 서버가 얼마나 잘 버티는지 보는 보고서입니다.
- `워커 수`는 동시에 실제 일을 처리하는 인원 수, `큐 크기`는 잠시 대기시켜 둘 수 있는 줄 길이라고 생각하면 됩니다.
- `성공`은 정상 처리된 요청 수, `503`은 서버가 너무 바빠서 거절한 요청 수, `오류`는 정상 응답 자체를 받지 못한 요청 수입니다.
- `평균 ms`보다 `p95 ms`가 더 중요합니다. 평균은 괜찮아 보여도 느린 요청이 가끔씩 길어질 수 있기 때문입니다.
- 가장 먼저 볼 곳은 `버스트 수준별 최적 조합` 표입니다. 여기에는 갑작스러운 몰림 상황에서 상대적으로 가장 안정적이었던 조합만 추려져 있습니다.

## 해석

- 이 스윕은 버스트 동시성만 따로 분리해서 봅니다. 일반 벤치 리포트와 달리 각 run의 요청 수는 동시 클라이언트 수와 같습니다.
- 각 조합을 10회 반복 측정했고, 성공/실패 수와 지연 시간 모두 평균과 표준편차를 같이 표시합니다.
- `혼합 (mixed)` 부하는 read lock 경로와 write lock 경로를 동시에 타기 때문에 우선적으로 해석해야 하는 지표입니다.
- `오류`가 0이 아니면, 포화 상태에서 클라이언트가 JSON 응답 자체를 받지 못하고 실패한 경우가 있었다는 뜻입니다. 이는 `503`만 세는 것보다 더 엄격한 기준입니다.
