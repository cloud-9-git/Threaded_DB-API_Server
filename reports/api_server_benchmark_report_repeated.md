# mini_db_server 벤치마크 리포트

## 처음 보는 분을 위한 안내

- 이 문서는 서버가 평소처럼 계속 요청을 받을 때 어떤 설정이 가장 무난한지 보는 보고서입니다.
- `워커 수`는 동시에 실제 일을 처리하는 인원 수, `큐 크기`는 잠시 대기시켜 둘 수 있는 줄 길이라고 생각하면 됩니다.
- `성공`이 높고 `503`과 `오류`가 낮을수록 좋습니다.
- `평균 ms`는 전체적인 속도, `p95 ms`는 느린 요청까지 포함했을 때의 체감 품질, `표준편차`는 같은 테스트를 여러 번 했을 때 결과가 얼마나 흔들리는지를 뜻합니다.
- 가장 먼저 볼 곳은 `최적 조합` 표입니다. 그 아래 `해석`과 `전체 결과`는 왜 그 조합을 골랐는지 설명하는 뒷받침 자료입니다.

## 해석

- 측정 전 인덱스를 사용하는 SELECT 1회를 먼저 보내 워밍업했기 때문에, 본 측정 지연 시간에는 최초 인덱스 생성 비용이 포함되지 않습니다.
- 각 조합을 10회 반복 측정했고, 표의 지연 시간과 성공/실패 수는 모두 평균과 표준편차를 같이 표시합니다.
- `503` 개수는 선택한 클라이언트 동시성에서 과부하가 걸렸을 때 반환된 응답 수입니다. 별도의 queue-full 검증으로 bounded queue 거절 경로도 따로 확인했습니다.
- 거절과 오류의 합이 같으면 p95 지연 시간이 더 낮은 조합을 우선 선택했고, 그마저 비슷하면 더 작은 워커/큐 조합을 우선 선택했습니다.
